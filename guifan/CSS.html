一、语法书写正确
注意：[1]z-index的属性值没有单位：例如：z-index:100;
[2]其他表述长宽高等的属性值都有单位：例如：margin:10px; height:24px; width:200px;等；
[3]其中只有属性值为0时可以不写单位
二、命名规则
[1]用字母、_号工、-号、数字组成，必须以字母开头，不能为纯数字。
[2]使用有意义的命名：
规则： （1）所有ID或者class字母和数字之间用“_”连接，命名不要用缩写
（2）导航：板块缩写名称_nav（例：简历板块：.jl_nav{}；社区板块：.sq_nav{}）
（3）其他位置：板块缩写名称_作用（例：找工作头部搜索部分：.zw_search{}）
页头：header 登录条：loginbar 标志：logo 侧栏：sidebar
广告：banner 子导航：subnav 菜单：menu 子菜单：submenu
搜索：search 滚动：scroll 页面主体：main 内容：content
标签页：tab 子导航：subnav 提示信息：msg 小技巧：tips
栏目标题：title 加入：joinus 指南：guild 服务：service
热点：hot 新闻：news 下载：download 注册：regsiter
状态：status 滚动：scroll 按钮：btn 投票：vote
合作伙伴：partner 友情链接：friendLink 页脚：footer 版权：copyright
[1] ID是用来标识具体模块，命名必须具体且唯一，由前缀和名字组成。不要滥用ID。如： #db-video-list。
[2] Class是用来标识某一类型的元素，命名简洁表意清楚。如：.list。
[3]推荐使用的class名：
表示状态 .on, .active, .selected, .hili
表示位置 .first, .last, .main, .side
表示结构 .hd, .bd, .ft, .col, .section
通用元素 .tb, .frm, .nav, .list, .item, .tag, .pic, .info
以上只是列出一些常用的简单的英文命名，常用类的命名要以常见英文单词为准，做到通俗易懂，并在适当的地方加以注释；

三、css代码优化（包括<1>语法优化<2>模块化，提高重用率）

1、css代码语法优化

1， Css命名不可以用html标签来命名，命名最好用约定俗成的缩写来命名，做到简单易懂
2，用优化的方式写css代码
例如：.zuixin{ width:350px; height:500px; border:1px solid #ccc;}
.retie{ width:350px; height:500px; border:1px solid #ccc;}
可优化为.zuixin,.retie{width:350px; height:500px; border:1px solid #ccc;}
3，符合单条CSS规则的书写格式要求
[1] 属性需要写在一行。需要在“{“和”}”前后加空格。 .selector { property:value;property:value; }
[3] 兼容多个浏览器时，将标准属性写在后面，如： -webkit-border-radius:4px;-moz-border-radius:4px;border-radius: 4px;
4，将作用于不同模块的CSS规则集中放在一起，同时用注释说明
注释的格式：/* mod: doulist */
通用规则同样分类放在一起。通用规则在具体模块规则的前面。如：/* button */
5， 尽量避免使用CSS Hack
推荐使用下面的：
区别属性：
IE6 _property:value
IE6/7 *property:value
IE6/7/8/9 property:value9
非IE6 property//:value
6， 使用after或overflow的方式清浮动
7， 避免使用低效的选择器
如：
    body > * {…}
    ul > li > a {…}
    #footer > h3 {…}
    ul#top_blue_nav {…}
    #searbar span.submit a { … }
9，尽量避免使用filter
10，不要直接修改标签的样式 如： div { … }
11，不要在标签上直接写样式 如：<div style=”margin-bottom:30px;”>
12， 不要在CSS中使用 expression
13， 不要在CSS中使用 @import
14， 不要在CSS中使用 !important
15， 绝对不要在CSS中使用 “*” 选择符

利用CSS将代码模块化，提高代码重用率

1、将各模块间出现类似的部分，提出来做为公共的定义，减少冗余：
base.css是全站基本样式。它需要放在所有CSS引用的最前面。它包括：网站所有头部、底部，标签reset、常用规则(链接、字体、边框、隐藏、清浮动等)、布局、各种模块基本样式（弹出层 的样式）等 这些公用属性写好后就不要再随意更改了，以免对其他地方产生影响，改动后，要经过全面测试。
2、将可以被重复调用的，提出来做为公共的定义，命名规则【以css属性名的首字母+属性值命名】
代码举例：
①文字样式
.f12{font-size:12px}
.f14{font-size:14px}
.fwb{font-weight:bold}
.fwn{font-weight:normal}
.tdu{text-decoration:underline}
.tdn{text-decoration:none}
②文字定位
.tl{ text-align:left}
.tr{ text-align:right}
.fl{ float:left; display:inline;}
.fr{ float:right; display:inline;}
③宽度、高度
.w100{ width:100px;}
.h100{ height:100px;}
④边距
.m10{ margin:10px;}
.mt20{margin-top:20px; }
3、利用css样式的优先级、继承性，用相近的css代码控制相近的样式，以减少用户控件的使用
例如：友情链接在A页面中是宽600px、背景颜色是蓝色；在B页面中是宽700px、背景颜色是红色，现在只用一个控件来实现
样式：
.box{ border:1px solid #ccc; marging:10px 0;}
. blue .box{ background:blue; width:600px;}
.red .box{ background:red; width:700px;}
用户控件：（可命名为youqing.ascx）
<div class="box">友情链接内容</div>
A页面html代码：
<div class="blue">调用用户控件</div>
B页面html代码：
<div class="red">调用用户控件</div>
