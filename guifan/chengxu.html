一、相关提示：

常用的两种规则解析:
　Camel命名规则：第一个词的首字母小写，其后单词的首字母全部大写 如： userName
　Pascal命名规则：所有单词的第一个字母都要大写，其余字母小写 如： UserName

二、详细内容：

(1)数据库方面的命名规范
数据库名：首先所起名称必须要有明确的含义，其次所起名称后面都要加Base作为标记，最后要遵循Camel命名规则
　　　　　例如：gcbBase 代表工程部数据库
数据库表名：首先所起名称必须要有明确的含义，其次所起名称前面都要加上数据库名称的缩写和下划线，最后要遵循Camel命名规则
　　　　　例如：gcb_userList 代表工程部数据库下面的员工信息表
数据库字段名：首先所起名称必须要有明确的含义，其次所起名称前面要加上数据库表名的缩写，中间用下划线隔开，最后要遵循Camel命名规则
　　　　　例如：u_name 代表工程部数据库下面员工信息表里面的员工姓名字段
存储过程命名：首先所起名称必须要有明确的含义，其次在所起名称之前要加上pro_作为标志，最后要遵循Camel命名规则
　　　　　例如：pro_getUserInfo 代表获取员工信息的存储过程
触发器名称：首先所起名称必须要有明确的含义，其次在所起名称之前要加上tr_作为标志，最后要遵循Camel命名规则
　　　　　例如：tr_addUserInfo 代表添加员工信息的触发器
在数据库语句编写时定义的变量的命名规范：首先所起名称必须要有明确的含义，其次要遵循Camel命名规则
　　　　　例如： userName 代表定义的变量是表示员工姓名的变量

(2)文件名、地址路径方面的命名规范
存储非全局图片的文件夹―img
存储播放文件的文件夹―flash
存储时间空间的文件夹―time+插件类型缩写 例如：timeTxt 代表文本形式的时间控件
存储数据库文件的文件夹―Base
存储着存储过程和触发器的文件夹―storeProcedure
文本编辑器文件夹的命名统一用默认的原名即可
存储着网站后台的文件夹名称推荐不使用Admin等公用命名
存储数据库或存储过程的文件夹中可以不放相应文件，但要标明所放文件地址
文件夹名称的命名规范遵照Camel命名规则，文件夹名称要有相应的含义
　　类文件名称的命名规范：
　　　　Model层中的文件名称=数据库表名+M 例如：gcb_userM.cs 员工表模型
　　　　BLL层的文件名称=数据库表名+B 例如：gcb_userB.cs 员工表逻辑操作类
　　　　PUB层的文件名称=有含义的名字+Pub 例如：cookiePub.cs 操作cookie的公共类
　　　　WEB层的文件名=有含义的名字 例如：userManage.aspx 员工信息管理页文件名称的命名规范遵照Camel命名规则，文件名称要有相应的含义,首页统一使用index.aspx 登录页统一用login.aspx
　　样式表中类文件的命名：普通样式文件的命名只要有含义即可，遵循Camel规则，对于自定义控件和模板页样式表的命名为：有含义的名字+_master 或者有含义的名字+_ascx 例如：userManger_master.cs
项目名称的命名规范：
　　解决方案名称：首先参照Pascal命名规范，其次名字要有一定的含义，在后面加上Project作为修饰 例如：GcbProject 代表的就是工程部的项目
　　解决方案下的各层命名：首先参照Camel命名规范命名，其次名字前面要加上解决方案名称的简写，最后加上各层特有的后缀标记(Web、BLL、Model、Pub) 例如：gcbWeb 显示层 gcbBLL 逻辑层 gcbModel 实力层 gcbPub 公共层

(3)页面控件的命名规范
控件名称=控件类型的缩写+有含义的名字 遵循Camel命名规则
常用控件名称的缩写：
Label前加 “lbl” LinkLabel前加 “llbl”Button前加 “btn”
TextBox前加 “txt”CheckBox前加 “chk”RadioButton前加 “rbtn”
Panel前加 “pnl”DataGridView前加 “dgv”ListBox前加 “lsb”
CheckedListBox前加 “clsb”TreeView前加 “tv”Calendar前加“cal”（Web）
Image前见“img”（Web）CheckBoxList前加“chkl”（Web）DataList前加“dl”（Web）
DropDownList前加“ddl”（Web）HyperLink前加“hlk”（Web）ImageButton前加“ibtn”（Web）
Table 前加“tb”（Web）TableCell前加“tbc”（Web）TableRow 前加“tr”（Web）
GridView前加“gdv”(Web)Repeater前加“rpt” (Web)HiddenField前加“hfld “(Web)
FileUpload前加“fup” (Web)Form 前加“frm”（WinForm）GroupBox前加 “grb”（WinForm）
PictureBox前加 “pic”（WinForm）ComboBox前加 “cbo”（WinForm）ListView前加 “lsv”（WinForm）
TabControl前加“tctl” （WinForm）Timer前加“tmr” （WinForm） 
(4)程序员编写的方法和定义的变量的命名规范
变量的命名遵循的是Camel命名规则，而且变量名称要有意义，局部变量必须在最接近使用时再声明，全局变量命名时不准使用(X、Y、Z)等简单无意义的名称(for 循环除外). 例如：userName 代表员工姓名的变量 特殊注意：Model层中的变量按照数据库中的字段名称命名；如果定义的变量类型为私有的，推荐在变量前加上下划线；
方法的命名规则遵照的是Pasca命名规则，而且名称要有相应的含义，普通方法的命名直接遵照规则既可 例如：GetUserInfo 获取用户信息的方法 对于像接口这样的方法需要在方法名前添加I标志 例如：IGetUserInfo 获取用户信息的接口

三、特殊要求：

1、 所有的页面事件都要加try catch 并且在事件外面加上注释 具体如下：
#region ZhangSan Add 11-03-18 添加员工信息
protected void btnAddUser_Click(object sender, ImageClickEventArgs e)
{
try
{

}
catch (Exception ex)
{
//跳转到友情提示页
}
}
#endregion
2、在页面中一律不准出现css和js等设定样式的代码。
3、所有的方法必须用 #region 注释扩起来 而且标明：姓名 Add 时间 例如：
#region ZhouXiaoTian Add 2011-03-18 添加员工信息
#endregion
如果连续很多方法都是同一人在同一天编写的 可以将注释括起他所写的所有方法
4、在编写BLL层中类下的方法时 按照方法所涉及到的主表 将编写的方法写到相应以主表命名的类中 例如：方法AddDept 添加地区方法 就要写到gcb_deptB.cs中
5、在编写代码时，如果涉及到要对代码进行格式转换时，要做好数据值的判断，防止因为数据原因而造成转换失败而引起的错误。
6、及时删除注释掉的大段代码，包括前后台
